# 元宝AI智慧学情系统 - 交互原型开发指令

## 项目背景
我正在准备腾讯元宝AI教育方向产品经理的面试，需要一个可交互的高保真产品Demo来展示我的产品设计方案。请基于附带的PRD文档和jsx原型参考文件，帮我构建一个完整的、可在浏览器中演示的交互原型。

## 产品定义
为元宝AI打造「智慧学情系统」，将教育功能从"单次问答"升级为"持续学习闭环"。目标用户是K12中小学生（应试场景）。

---

## 技术要求

### 框架与工具
- 使用 React（单文件组件），或 HTML + Tailwind CSS + vanilla JS
- 移动端优先设计（宽度 430px 模拟手机屏幕），桌面端居中展示
- 不依赖后端，所有数据用前端 mock
- 要求可以直接在浏览器中打开演示

### 设计风格
- **整体风格**：模拟元宝AI的真实UI风格——简洁、现代、圆角卡片、渐变色头部
- **主色**：#4F6EF7（元宝蓝）辅助色 #FF6B35（橙色强调）
- **字体**：-apple-system, "PingFang SC", sans-serif
- **卡片**：圆角16px，轻投影，白底
- **动画**：页面切换有过渡动画，卡片入场有 stagger 延迟动画，按钮有 hover/active 反馈
- **顶部**：渐变色头部区域（蓝色渐变到页面背景色），显示用户信息和快捷统计
- **底部**：固定 Tab Bar，4个标签页，带毛玻璃效果

---

## 四大功能模块详细说明

### Tab 1：📝 智能错题本

#### 页面结构
1. **顶部统计区**：三个统计卡片横排——待复习（红色）、复习中（黄色）、已掌握（绿色），数字要有计数动画
2. **筛选栏**：横向滚动的学科筛选按钮（全部/数学/英语/物理/语文），选中态有蓝色高亮
3. **错题卡片列表**：每张卡片包含：
   - 左侧彩色竖条标识错因等级
   - 错因标签（表层错误/蓝色、概念迷雾/橙色、策略偏差/红色、思维惯性/紫色）
   - 学科 + 知识点标签
   - 题目文本
   - "你的答案"（红底）vs "正确答案"（绿底）双栏对比
   - 两个按钮：「查看错因」展开错因分析详情、「✨ AI引导讲解」进入苏格拉底对话

#### 苏格拉底式AI引导讲解（核心亮点，必须做好）
点击「AI引导讲解」后进入聊天界面：
- 模拟微信风格的对话气泡：AI在左（白底+蓝色标签"✨ 元宝AI导师"）、用户在右（蓝色渐变底）
- **对话逻辑**：AI不直接给答案，而是一步步反问引导——
  - AI："这道题考查的是哪个知识点？"
  - 用户点击选项回答
  - AI："很好！那如果把点代入公式，你会得到什么？"
  - 用户点击选项回答
  - AI："注意移项时的正负号哦 😊"
  - 用户点击得出正确答案
  - AI："🎉 太棒了！你之前的错误是xxx。这道题已标记为已理解，3天后会复习巩固 ✅"
- 用户侧不需要手动输入，而是点击**预设的回答按钮**推进对话
- AI回复前要有**打字中动画**（三个跳动的圆点）
- 对话完成后显示「✅ 已掌握，返回错题本」按钮
- 页面顶部有「← 返回」按钮

#### Mock数据要求
准备至少4道错题，覆盖数学（二次函数、概率）、物理（牛顿定律）、英语（定语从句），每道题包含完整的题目、错误答案、正确答案、错因分析文本

### Tab 2：📊 学情诊断

#### 页面结构
1. **综合评分卡**：蓝色渐变大卡片，显示总分（如74/100），附带"较上周↑3分"和"超过67%同年级学生"
2. **四科掌握度**：2x2 网格卡片（数学📐/英语🔤/物理⚡/语文📖），每科显示：
   - 学科名+emoji
   - 进度条（颜色与学科对应）
   - 掌握度百分比
   - 错题数量
   - 可点击切换下方知识点详情
3. **知识点掌握排行**：选中某学科后展示该学科下的知识点列表
   - 按掌握度从低到高排列
   - 每行：知识点名 + 进度条（<50%红色/<70%橙色/>70%绿色）+ 百分比 + 趋势箭头（📈📉➡️）
   - 顶部标注"X个薄弱点"
4. **AI诊断建议**：黄色渐变底的建议卡片，文字说明当前最薄弱的知识点和具体建议

#### 数据要求
- 数学知识点：二次函数(45%)、概率统计(55%)、三角函数(78%)、立体几何(62%)、数列(85%)、导数(38%)
- 其他学科也准备3-5个知识点数据

### Tab 3：🎯 个性化复习

#### 页面结构
1. **记忆保持曲线可视化**：
   - 用 SVG 绘制两条曲线——红色虚线"不复习"（快速下降）、蓝色实线"定时复习"（锯齿形回升）
   - 蓝色曲线上标注3个复习节点（蓝色圆点）
   - 底部时间轴：今天→1天后→3天后→7天后→30天后
   - 图例说明
2. **今日任务清单**：
   - 每个任务卡片包含：时间标签、任务描述、任务类型图标
   - 四种任务类型：🔄间隔复习（红色底）、✏️巩固练习（橙色底）、📖拓展阅读（蓝色底）、📝综合测验（绿色底）
   - 紧急任务有橙色"紧急"标签
   - 点击展开任务详情+「开始」按钮
   - 详情文字要有教育专业感，说清楚为什么在这个时间做这个任务
3. **本周复习成果**：绿色渐变底卡片，显示复习次数、掌握错题数、连续学习天数（带🔥）

#### 任务数据
- 08:00 复习二次函数顶点式（3题）- 紧急
- 12:30 巩固条件概率基础（2题）
- 18:00 微信读书推荐章节阅读
- 21:00 本周错题随机抽测（5题）- 紧急

### Tab 4：🔗 微信生态串联

#### 页面结构
1. **全链路流程图**：纵向流程，5个步骤节点——
   📸拍题答疑（蓝色渐变，标注"现有功能"）→ 📝错题沉淀 → 📊学情分析 → 🎯复习计划 → 💬微信提醒
   每个节点之间用箭头连接
2. **四大生态联动卡片**（可点击展开详情）：
   
   **💬 微信提醒推送**：展开后模拟一条微信服务号消息推送，包含元宝logo、推送标题、提醒文本"你有3道二次函数错题即将到达最佳复习时间"、"点击立即开始复习→"
   
   **📖 微信读书联动**：展开后显示2本推荐书籍卡片，每本包含书名、推荐章节、知识点匹配度百分比
   
   **👥 学习小组PK**：展开后显示PK排行榜（3人：小明95分#1、你82分#2高亮、小红78分#3），底部文字"在微信群@元宝发起今日数学PK赛"
   
   **📄 腾讯文档导出**：展开后显示三个导出选项卡（错题本PDF📋、学情报告📊、复习计划表📅）+「导出到腾讯文档→」按钮

## 交互细节要求

1. **Tab切换**：底部Tab点击切换，选中态icon放大、文字变蓝加粗，带过渡动画
2. **卡片入场**：使用 stagger 动画，每张卡片延迟 0.08s 依次入场（从下往上淡入）
3. **数字动画**：关键数字（总分74、各种统计数字）页面进入时有从0递增的计数动画
4. **展开/收起**：错题详情、任务详情、生态卡片详情均为点击展开/收起，带平滑动画
5. **苏格拉底对话**：
   - AI消息出现前有 0.8-1s 的打字动画
   - 用户点击按钮后消息立即出现，随后触发AI回复
   - 消息列表自动滚动到最新消息
6. **滚动**：页面内容可自然滚动，底部Tab固定不动
